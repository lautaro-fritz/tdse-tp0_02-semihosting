Nota de Seguridad: El uso de printf(p) es una vulnerabilidad conocida como "Format String Attack". Si el usuario escribe algo como %x, el programa intentará leer datos de la memoria. Lo correcto sería printf("%s", p).

2. ¿Qué es Semihosting?
El Semihosting es un mecanismo que permite que un programa ejecutándose en un microcontrolador (el "objetivo" o target) utilice las capacidades de entrada y salida de la computadora que lo está depurando (el "anfitrión" o host).

Normalmente, un microcontrolador no tiene una pantalla ni un teclado. El semihosting permite que:

printf() escriba directamente en la consola del IDE (como STM32CubeIDE o Keil).

scanf() lea lo que el programador escribe en el teclado de su PC.

Incluso se puedan leer o escribir archivos directamente en el disco duro de la computadora desde el código del microcontrolador.

Relación con el código fuente
La relación es directa y necesaria para que el código funcione:

initialise_monitor_handles(): Esta función pertenece a la librería de semihosting de ARM. Sin ella, el programa se congelaría al intentar ejecutar el primer printf.

Redirección de E/S: Aunque el código inicializa el periférico USART2, las funciones printf y scanf en este archivo no están enviando datos por los pines TX/RX físicos, sino que están enviando interrupciones al depurador (vía ST-LINK o J-Link) para que este gestione la entrada y salida de texto.

Librería Estándar: El uso de <stdio.h> en sistemas embebidos suele requerir una configuración de "bajo nivel" para saber a dónde enviar los bytes; el semihosting es la forma más rápida de lograrlo sin configurar manualmente el puerto serie.
